Get up and running with Dashi in just a few minutes. This guide walks you through launching the application and having your first conversation using voice or text input.

<Steps>
  <Step title="Set up environment variables">
    
    Dashi requires API keys for AI services. Create a `.env` file in your project root:

    ```bash
    # In the project root directory
    touch .env
    ```

    Add your required API keys to the `.env` file:

    ```env
    # Groq API key for speech-to-text transcription
    GROQ_API_KEY=your_groq_api_key_here
    ```

    <Tip>
      Get your Groq API key from [console.groq.com](https://console.groq.com). The `whisper-large-v3` model is used for transcription.
    </Tip>

  </Step>

  <Step title="Start the development server">

    Launch Dashi in development mode to start both the frontend and backend:

    ```bash
    npm run dev
    # or
    bun dev
    ```

    This command starts:
    - The Next.js development server on `http://localhost:3000`
    - The Tauri application window with your Dashi interface
    - The Rust backend with audio processing capabilities

    The Dashi window will open automatically, showing the animated mascot character on a dark background.

  </Step>

  <Step title="Use voice mode">

    Experience voice input with Dashi:

    1. Ensure your microphone is connected and permissions are granted
    2. Press and hold <strong>Cmd+Opt+V</strong> (Mac) or <strong>Ctrl+Alt+V</strong> (Windows/Linux)
    3. Speak your message clearly
    4. Release the keys to stop recording
    5. Watch Dashi transcribe your speech and generate a response

    <CodeGroup>
    ```typescript dashimain.tsx
    // Voice recording is handled by the useVoiceRecorder hook
    const { startRecording, stopAndTranscribe, isRecording } = useVoiceRecorder();

    // Press Cmd+Opt+V to trigger:
    await startRecording();  // Starts microphone capture

    // Release Cmd+Opt+V to trigger:
    const transcription = await stopAndTranscribe();  // Stops and transcribes
    ```
    ```rust audio.rs
    // Rust backend processes audio and calls Groq API
    #[tauri::command]
    pub async fn transcribe(audio: Vec<f32>) -> Result<String, String> {
        let wav_data = process_audio(audio)?;
        let request = SpeechToTextRequest::new(wav_data)
            .model("whisper-large-v3")
            .language("en");
        let result = client.speech_to_text(request).await?;
        Ok(result.text)
    }
    ```
    </CodeGroup>

    The mascot animates and music notes appear around Dashi while recording. Your transcribed text is converted to a WAV file and sent to Groq's API for transcription (src/components/ui/dashimain.tsx:198-274).

  </Step>

  <Step title="Use chat mode">

    For text-based conversations, switch to chat mode:

    1. Press <strong>Cmd+Opt+T</strong> (Mac) or <strong>Ctrl+Alt+T</strong> (Windows/Linux)
    2. The chat interface appears with a text input field
    3. Type your message in the input field
    4. Click **Send** or press Enter
    5. Watch Dashi's response stream in real-time

    <CodeGroup>
    ```typescript dashimain.tsx
    // Chat interface handles user messages and AI responses
    const handleChatSubmit = async (e: React.FormEvent) => {
      e.preventDefault();
      if (chatInput.trim()) {
        setChatMessages(prev => [...prev, { text: chatInput, isUser: true }]);
        
        const dashiResponseStream = await generate_response(chatInput);
        for await (const chunk of dashiResponseStream) {
          if (chunk.content && chunk.content !== "undefined") {
            setChatMessages(prev => {
              const updated = [...prev];
              updated[updated.length - 1] = {
                ...updated[updated.length - 1],
                text: (updated[updated.length - 1].text || "") + chunk.content
              };
              return updated;
            });
          }
        }
      }
    };
    ```
    ```typescript letta.ts
    // Letta client generates streaming AI responses
    export const generate_response = async (input: string) => {
      const stream = await letta_client.agents.messages.createStream(agent_id, {
        messages: [{ role: "user", content: input }],
        streamTokens: true,
      });
      return stream;
    };
    ```
    </CodeGroup>

    Messages appear in a chat bubble interface. User messages show in teal (#8CEBE5) on the right, while Dashi's responses appear in purple (#D500D8) on the left (src/components/ui/dashimain.tsx:450-586).

  </Step>

  <Step title="Reset to default mode">

    To exit voice or chat mode and return Dashi to its default state:

    Press the <strong>Escape</strong> key at any time.

    This resets the interface, closes the chat window, stops any active recordings, and returns Dashi to its initial hovering position with low opacity (src/components/ui/dashimain.tsx:130-145).

  </Step>
</Steps>

## What's happening behind the scenes

When you use voice mode:

1. Browser captures audio via MediaRecorder API
2. Audio is converted to WAV format (16kHz mono) in Rust
3. WAV data is sent to Groq's Whisper model for transcription
4. Transcribed text is passed to Letta AI agent
5. Response streams back in real-time to the UI

When you use chat mode:

1. Your message is sent directly to the Letta AI agent
2. Agent processes the input and generates a response
3. Response streams token-by-token to the React frontend
4. Chat interface updates in real-time as tokens arrive

## Common issues

<Warning>
    **Microphone access denied**: Ensure your browser or system has granted microphone permissions to Dashi. Check your system privacy settings.
</Warning>

<Warning>
    **GROQ_API_KEY not set**: Verify your `.env` file exists in the project root and contains a valid Groq API key. The Rust backend reads this environment variable at runtime.
</Warning>

<Warning>
    **Letta connection failed**: Ensure your Letta agent service is running at `http://localhost:8283`. The agent ID is configured in `src/lib/letta.ts`.
</Warning>
